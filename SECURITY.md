# Security Policy

## Reporting a Vulnerability

Do not open public issues for suspected vulnerabilities.

Use GitHub private vulnerability reporting or open a private repository security advisory. Include:

- affected component or workflow;
- reproduction steps;
- observed impact;
- relevant logs with secrets removed.

## Supported Version

Security fixes target the `main` branch.

## Secrets

Never include Telegram tokens, Google credentials, service account keys, webhook channel tokens, or production identifiers in issues, pull requests, logs, screenshots, or documentation examples.
